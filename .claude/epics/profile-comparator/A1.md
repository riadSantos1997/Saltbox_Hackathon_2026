# A1 · Salesforce Client + OAuth Session Helper

## 🎯 Objective
Build a thin wrapper around the Salesforce REST/Tooling APIs that reads session tokens for both orgs (assumed already connected to S1) and exposes a single `query(soql, org)` function the rest of Stream A and Stream D can build on.

## 📂 Files
- `lib/salesforce/client.ts` — client factory + `query()` + error handling
- `lib/salesforce/types.ts` — **contract file**, commit by 0:30 (blocks B1)

## ✅ Acceptance Criteria
- [ ] `getClient(org: 'A' | 'B')` returns an authed client with base URL `https://{orgDomain}/services/data/vXX.0/`.
- [ ] `query(soql, org)` posts to Tooling API endpoint `/tooling/query/?q=` and returns typed results.
- [ ] 401/403 responses surface a distinct `SessionExpiredError` so D3 can render "Session expired. Reconnect Org X."
- [ ] 10-second timeout on every request (addresses PRD high-severity risk).
- [ ] `lib/salesforce/types.ts` exports `OrgId`, `PermissionCategory`, `PermissionRow`, `ScrapeResult` per the epic contract — committed to git by 0:30.

## 🔗 Dependencies
- **Depends on:** nothing (can start at 0:20).
- **Blocks:** A2, A3, A4 (all need `client.ts`); B1 (needs `types.ts`); D2 (needs client for tool calls).

## 📝 Implementation Sketch
```ts
export type OrgId = 'A' | 'B';
export class SessionExpiredError extends Error { constructor(public org: OrgId) { super(); } }

export function getClient(org: OrgId) {
  const { accessToken, domain, apiVersion } = readSession(org); // from S1 connection layer
  return {
    query: async <T>(soql: string): Promise<T> => {
      const url = `https://${domain}/services/data/${apiVersion}/tooling/query/?q=${encodeURIComponent(soql)}`;
      const res = await fetch(url, {
        headers: { Authorization: `Bearer ${accessToken}` },
        signal: AbortSignal.timeout(10_000),
      });
      if (res.status === 401 || res.status === 403) throw new SessionExpiredError(org);
      if (!res.ok) throw new Error(`SF query failed: ${res.status}`);
      return res.json();
    },
  };
}
```

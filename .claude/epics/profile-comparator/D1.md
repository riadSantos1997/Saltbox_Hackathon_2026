# D1 · Prompt Parser

## 🎯 Objective
Extract `{profileName, orgA, orgB}` from free-form user text. If only 1 org found, trigger the clarifying-question branch. Implements **AC-01**.

## 📂 Files
- `lib/ai/parser.ts` — `parsePrompt(text): ParsedPrompt | NeedsClarification`
- `types/shared.ts` — cross-stream types owned by D

## ✅ Acceptance Criteria (→ AC-01)
- [ ] Correctly extracts profile name + both org names from "Compare the Sales Rep profile between OrgA and OrgB" style prompts.
- [ ] Org A is always the first-mentioned / designated as reference.
- [ ] Single-org prompts return `{ needsClarification: 'missing_second_org' }` → D3 renders a clarifying question.
- [ ] Profile name preserves original casing (downstream A2 handles case-insensitive match).
- [ ] Returns structured result, not a best-guess string.

## 🔗 Dependencies
- **Depends on:** nothing (can start at 0:20 in parallel with A/C).
- **Blocks:** D3.

## 📝 Implementation Note
A rule-based extractor with a couple regex patterns is plenty for the demo. Only reach for an LLM-based extractor if time allows — simplicity wins the sprint.

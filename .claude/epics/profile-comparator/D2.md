# D2 · Vercel AI SDK Tool Definitions

## 🎯 Objective
Define the three Vercel AI SDK `tool()` handlers that the LLM calls mid-conversation: `validateProfile`, `listObjects`, `runComparison`. Each is a thin bridge to a Stream A endpoint.

## 📂 Files
- `lib/ai/tools.ts`

## ✅ Acceptance Criteria
- [ ] `validateProfile({ profileName, orgA, orgB })` → calls `/api/salesforce/profiles` → returns `found | suggestions | missingIn`.
- [ ] `listObjects({ org })` → calls `/api/salesforce/objects` → returns `{apiName, label}[]`.
- [ ] `runComparison({ category, profileName, selectedObjects?, orgA, orgB })` → calls `/api/salesforce/scrape` then forwards to `/api/export` → returns `{ downloadUrl }` or `{ noDifferences: true }`.
- [ ] Each tool has Zod input schema.
- [ ] Errors bubble up with user-facing messages (e.g., `SessionExpiredError` → "Session expired. Reconnect Org X.").

## 🔗 Dependencies
- **Depends on:** A1 (to know endpoint contracts and error types).
- **Blocks:** D3.

## 📝 Notes
During parallel dev, stub the fetch responses with fixture data until A2/A3/A4 endpoints land — this lets D3 progress without waiting.

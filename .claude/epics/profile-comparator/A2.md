# A2 · Profile Lookup + Fuzzy Match

## 🎯 Objective
Given a profile name string and two org IDs, validate the profile exists in both orgs. On miss, return up to 5 fuzzy-matched suggestions from the union of both orgs. Implements **F-01 / AC-02**.

## 📂 Files
- `lib/salesforce/profiles.ts` — pure query + scoring logic
- `app/api/salesforce/profiles/route.ts` — thin HTTP wrapper consumed by Stream D tool

## ✅ Acceptance Criteria (→ AC-02)
- [ ] Case-insensitive exact match against `Profile.Name` AND `Profile.Label` (addresses Medium risk: API name vs. label ambiguity).
- [ ] If exact match found in both orgs → return `{ found: true, profile: {...} }`.
- [ ] If match in one org only → return `{ found: false, missingIn: 'A' | 'B' }` — D3 halts the flow.
- [ ] If neither exact → return `{ found: false, suggestions: [...up to 5...] }` within 2s.
- [ ] Fuzzy scoring = simple Levenshtein or substring — doesn't need to be clever.

## 🔗 Dependencies
- **Depends on:** A1 (`lib/salesforce/client.ts` + `types.ts`).
- **Blocks:** D2 (tool calls this endpoint).

## 📝 Implementation Sketch
```ts
// SOQL (Tooling API): SELECT Id, Name, Label FROM Profile
const results = await Promise.all([
  clientA.query<{records: Profile[]}>(`SELECT Id, Name, Label FROM Profile`),
  clientB.query<{records: Profile[]}>(`SELECT Id, Name, Label FROM Profile`),
]);
// exact match (case-insensitive on name OR label)
// else fuzzy score against union, return top 5
```
